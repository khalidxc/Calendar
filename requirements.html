KFUPM Calendar Requirements

Functional Requirement

  1. The system should let the student and the instructor access his calendar in three listed views: Day, Week and Month.
  2. The system should let the student and the instructor add events, delete events and view them.
  3. The system should let the instructor check and find the best day for assigning a major, a quiz , homework and any other events type.
  4. The system should let the student accept or reject any events created by his colleagues.
  5. The system should send a notification to any user if any event assigned to him.
  6. The system should let the instructor view his sections students events except the personal ones.


Non-Functional Requirement

  1. Availability, so the system will be available in 7*24 except Friday from 5:00 AM to 5:15 for backup proccess.
  2. Reliability, the system should be implemented using the last reliable technologies.
  3. Privacy, no user can view others' personal events, also all the data should be encrypted.
  4. Efficiency, so any static elements will be stored in the cache, also every page will be loaded in less than half a second.
